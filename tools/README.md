# Tolerance Data Import Tools

Tools for populating `parameters.db` with tolerance values (min_nominal, max_nominal).

## Overview

The database `orbita/config/address/parameters.db` has 383 parameters but all `min_nominal` and `max_nominal` values are currently NULL. These tools help import actual tolerance values from source documents.

## Tools

### 1. `extract_xlsx.py` - Extract data from Excel

Reads `посмотри/о БД/База_данных_финальная.xlsx` and dumps all data to CSV.

Handles:
- Inline strings (used in this xlsx)
- Shared string references
- Numeric cells

**Usage:**
```bash
python3 tools/extract_xlsx.py
```

**Output:**
- `tools/xlsx_dump.csv` - Complete dump of xlsx data
- Console output shows:
  - Column headers
  - Columns with data
  - First 5 rows
  - Whether tolerance columns were found

**Note:** This xlsx contains the address mappings but does NOT have tolerance values (min/max/nominal columns). It's used for reference only.

### 2. `import_tolerances.py` - Import CSV data into DB

Imports tolerance values from a CSV file into `parameters.db`.

**CSV Format (required columns):**
```
full_address,min_nominal,max_nominal[,units]
```

Where:
- `full_address`: The parameter's full address string (e.g., "M16П2A80B80C11D42E12")
  - Can be blank if you provide address components
- `min_nominal`: Minimum tolerance value (numeric)
- `max_nominal`: Maximum tolerance value (numeric)
- `units` (optional): Unit of measurement (e.g., "В", "А", "°С")

Alternatively, you can omit `full_address` and provide address components:
```
stream,a,b,c,d,e,x,t,p,min_nominal,max_nominal[,units]
```

**Matching Strategy:**
1. First tries to match by `full_address` (case/whitespace insensitive)
2. If not found, constructs a component key from (stream, a, b, c, d, e, x, t, p) and matches by that
3. Reports all unmatched addresses

**Usage:**
```bash
# Dry-run (default): shows what would be updated without changing DB
python3 tools/import_tolerances.py tolerances.csv

# Also explicit
python3 tools/import_tolerances.py tolerances.csv --db orbita/config/address/parameters.db --dry-run

# Apply changes to database
python3 tools/import_tolerances.py tolerances.csv --apply

# Custom database path
python3 tools/import_tolerances.py tolerances.csv --db /path/to/parameters.db --apply
```

**Output:**
- Match statistics (total, matched, not found, match rate %)
- List of unmatched addresses
- In dry-run mode: preview of first 3 updates (param_id, min, max, units)

## Workflow

1. **Extract source data:**
   - Run `extract_xlsx.py` to see what's in the xlsx
   - Check `посмотри/о телеметрии орбита/Приложение 2 (цифра).docx` for tolerance specifications
   - The docx has 18 tables (6 streams × 3 document versions) with parameters and periods/frequencies, but NOT min/max values

2. **Prepare CSV with tolerances:**
   - Manually extract tolerance values from the docx or other source
   - Create a CSV file with columns: `full_address, min_nominal, max_nominal[, units]`
   - Example:
     ```csv
     full_address,min_nominal,max_nominal,units
     M16П2A80B80C11D42E12,-5.0,5.0,В
     М16П1A11B21T21,0.0,100.0,
     M16П2A80B80C11D22E12T05P01,-10,10,°С
     ```
   - Or with components:
     ```csv
     stream,a,b,c,d,e,x,t,p,min_nominal,max_nominal,units
     П2,80,80,11,42,12,,,,−5,5,В
     П1,11,21,,,,,21,1,0,100,
     ```

3. **Dry-run import:**
   ```bash
   python3 tools/import_tolerances.py your_tolerances.csv --dry-run
   ```
   - Check the match rate
   - Review any unmatched addresses
   - Verify the update preview

4. **Apply to database:**
   ```bash
   python3 tools/import_tolerances.py your_tolerances.csv --apply
   ```

## File Reference

- `extract_xlsx.py` - Xlsx parsing utility (zipfile + xml.etree, no dependencies)
- `import_tolerances.py` - CSV → DB importer (sqlite3, csv modules)
- `README.md` - This file
- `xlsx_dump.csv` - Generated by extract_xlsx.py (for reference)
- `tolerances_template.csv` - Template for preparing tolerance CSV (auto-generated if no data found)

## Database Schema

```sql
-- parameters table
CREATE TABLE parameters (
    id INTEGER PRIMARY KEY,
    name TEXT,
    category TEXT,
    signal_type TEXT,
    units TEXT,
    min_nominal REAL,           -- Updated by import_tolerances.py
    max_nominal REAL            -- Updated by import_tolerances.py
);

-- addresses table (maps to parameters)
CREATE TABLE addresses (
    id INTEGER PRIMARY KEY,
    param_id INTEGER,           -- Foreign key to parameters.id
    is_zu INTEGER DEFAULT 0,
    stream TEXT,                -- From address components
    a TEXT,
    b TEXT,
    c TEXT,
    d TEXT,
    e TEXT,
    x TEXT,
    t TEXT,
    p TEXT,
    informativnost INTEGER,
    full_address TEXT           -- Used for matching
);
```

## Notes

- **No dependencies:** Uses only Python standard library (zipfile, xml.etree, sqlite3, csv, argparse)
- **Idempotent:** Running with `--apply` multiple times on the same CSV is safe (overwrites previous values)
- **Dry-run by default:** To prevent accidental database modifications, `--dry-run` is the default mode
- **Tolerance source:** Values must come from `Приложение 2 (цифра).docx` or provided by the user
- **Case-insensitive matching:** Full address matching is case/whitespace insensitive
