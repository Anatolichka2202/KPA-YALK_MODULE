# Слоистые допуски (Tolerance Layers)

## Описание

Система допусков в Orbita Desktop поддерживает трёхуровневую иерархию, позволяющую гибко управлять пределами нормы для каждого канала:

### Уровни приоритета (от высшего к низшему)

1. **Override слой (Сессионный)** — динамические допуски, устанавливаемые во время сеанса (например, через API или пользовательский интерфейс)
2. **Config слой (.tol файл)** — статические допуски, загруженные из файла `address/tolerances.tol`
3. **Database слой (БД)** — базовые допуски из таблицы `parameters` (столбцы `min_nominal`, `max_nominal`)

Если допуск не найден на одном уровне, система автоматически обращается к следующему (более низкому) уровню.

## Формат файла tolerances.tol

Файл расположен в каталоге `address/` рядом с исполняемым файлом OrbitaDesktop.

**Формат строк:**
```
<АДРЕС> <LO> <HI> [<NOMINAL>]
```

Где:
- `<АДРЕС>` — стандартный адрес канала (например, `M16П2A40B40C10D10E11T21` или `P2A40B40C10D10E11T21`). Внутренняя нормализация игнорирует префикс `M<N>`.
- `<LO>` — минимальное значение допуска (сырой код АЦП)
- `<HI>` — максимальное значение допуска (сырой код АЦП)
- `<NOMINAL>` — (опционально) номинальное значение. Если не указано, вычисляется как `(LO + HI) / 2`

**Синтаксис:**
- Пустые строки пропускаются
- Строки, начинающиеся с `#`, — комментарии
- Разделитель — один или несколько пробелов/табуляций

**Пример:**
```
# Допуски канала ЯТКБСУ (stream 2, вибрация)
M16П2A40B40C10D10E11T21 200 800 500

# Быстрые датчики
P1A70B12C10D10T01 510 742 626
```

## Интеграция в код

Резолвер представлен классом `ToleranceResolver`:

```cpp
// Сеттеры
resolver.setDb(dbProvider);
resolver.loadConfigFile(QString(".../address/tolerances.tol"));
resolver.setOverride(QString("M16П1A40..."), tolerance_t);

// Получение эффективного допуска
chstatus::Tolerance tol = resolver.resolve(QString("M16П1A40..."));
```

## Процесс загрузки

При запуске приложения:
1. MainWindow инициализирует `ToleranceResolver`
2. Устанавливается БД как источник (Database слой)
3. Загружается файл `tolerances.tol` в Config слой
4. Резолвер передаётся всем виджетам (MainPage, DetailView и дочерним компонентам)

Если файл `tolerances.tol` отсутствует — логируется предупреждение (код -1), но приложение работает корректно (используются Database и Override слои).

## Типичные сценарии использования

### Config слой
Используйте для **стационарных допусков** заводских приборов, которые не меняются во время сеанса:
```
# Гироскоп канала 1
M16П1A40B40C10D10E11T21 100 900 500
```

### Override слой
Используйте программно при необходимости **временно изменить допуск** (например, для диагностики или специальных испытаний):
```cpp
chstatus::Tolerance testTol;
testTol.set = true;
testTol.lo = 50;
testTol.hi = 950;
testTol.nominal = 500;
toleranceResolver.setOverride(address, testTol);
```

## Отладка

Проверить загруженные допуски можно через:
1. Визуальное отображение в таблице (Макет B, колонка "Допуск")
2. Логирование в консоль (будут выведены адреса и значения)
3. Инспектирование файла `address/tolerances.tol` в текстовом редакторе
