1. Общая схема
Вход: аналоговый сигнал с АЦП (12‑битные отсчёты, 0…4095).
Выход: цифровой поток битов → 12‑битные слова → фразы → группы → циклы → кадры.
Режимы информативности: M16, M08, M04, M02, M01 (определяют плотность данных).

Основные этапы:

Пороговая обработка – превращение аналоговых значений в биты (0/1).

Поиск маркеров – синхронизация (маркеры фраз, групп, циклов, кадров).

Сбор слов – из битов формируются 12‑битные слова.

Группировка – слова объединяются во фразы, группы, циклы.

Извлечение параметров – по адресной системе (T01, T05, T21, …).

2. Пороговая обработка (битовый поток)
Для каждого отсчёта АЦП value вычисляется порог porog:

cpp
porog = (maxValue + minValue) / 2;   // по всем отсчётам текущего буфера
Битовое значение bit получается сравнением с порогом:

cpp
bit = (value >= porog) ? 1 : 0;
В зависимости от настройки (инверсия) бит может быть инвертирован.

Полученные биты складываются в кольцевой буфер fifoMas размером FIFOSIZE = 2 500 000.

3. Структура телеметрии (общая)
Слово – 12 бит (биты нумеруются от старшего 11 до младшего 0).

Фраза – 16 слов (для M16). Для M08/M04/M02/M01 число слов во фразе равно 2 × информативность (т.е. 16, 8, 4, 2 слова).

Группа – 128 фраз (нумерация 1…128) → содержит 128 × 16 = 2048 слов для M16.

Цикл – 32 группы (нумерация 1…32) → 32 × 2048 = 65536 слов.

Кадр – 4 цикла (нумерация 1…4). Кадр – это полный период сбора.

Таким образом:

M16: 2048 слов/группа, 65536 слов/цикл, 262144 слова/кадр.

M08: 1024 слов/группа, 32768 слов/цикл.

M04: 512 слов/группа, 16384 слов/цикл.

M02: 256 слов/группа, 8192 слов/цикл.

M01: 128 слов/группа, 4096 слов/цикл.

4. Алгоритм для M16 (режим 16 слов во фразе)
4.1 Поиск первого маркера фразы
Функция SearchFirstFraseMarker проверяет биты на определённых позициях относительно текущего положения в буфере.

Проверяются 15 позиций (смещения в битах):

text
0, 24, 48, 72, 96, 120, 144, 168, 216, 240, 264, 288, 312, 336, 360
Эталонная последовательность (ожидаемые биты):

text
0, 1, 1, 1, 1, 0, 0, 0, 1, 0, 0, 1, 1, 0, 1
Если все совпадают – маркер найден. После этого задаётся расстояние до следующего маркера фразы:
pointCount = 383 бита.

4.2 Разбор фразы
После нахождения маркера начинается сбор 12‑битных слов:

Из кольцевого буфера последовательно читаются биты (по одному).

Каждые 12 бит образуют слово.

Слово обрабатывается функцией WordProcessing.

4.3 Обработка слова (WordProcessing)
Слово записывается в массив группы masGroup[wordIndex] (младшие 11 бит) и masGroupAll (все 12 бит).

Отслеживаются номера:

wordNum – от 1 до 16.

fraseNum – от 1 до 128 (увеличивается после каждого 16‑го слова).

groupNum – от 1 до 32 (увеличивается после 128 фраз).

ciklNum – от 1 до 4 (увеличивается после 32 групп).

Маркер кадра:
если wordNum == 1 && fraseNum == 16 && groupNum == 1 и старший бит слова (12‑й) = 1, то фиксируется начало кадра.

Маркеры группы и цикла собираются из чётных фраз (2,4,6,…,14) по 12‑му биту (бит 11, если считать с 0).
Накопленный 8‑битный код сравнивается:

114 (01110010) → маркер группы.

141 (10001101) → маркер цикла.

При нахождении маркера группы увеличивается groupNum, при маркере цикла – ciklNum (и сброс группы на 1).

Когда группа полностью заполнена (2048 слов), вызывается OutDate – вывод данных на графики и в файл.

Запись в файл .tlm происходит после заполнения целого цикла (65536 слов) – данные копируются в masCircle и затем сохраняются.

5. Алгоритм для M08/M04/M02/M01 (разная информативность)
Здесь другой метод синхронизации, основанный на поиске маркеров фразы как периодических импульсов.

5.1 Константы (зависят от информативности)
Режим	Слов во фразе	widthPartOfMF	markKoef	minSizeBetweenMrFrToMrFr
M08	8	3	6.357828776	1220
M04	4	6	12.715657552	1220
M02	2	12	25.431315104	1220
M01	1	25	50.862630020	1220
widthPartOfMF – длина маркера (количество единиц и нулей в маркере).

markKoef – коэффициент для определения середины маркера.

minSizeBetweenMrFrToMrFr – ожидаемое расстояние между маркерами в точках (битах).

5.2 Поиск маркера фразы (FindFraseMark)
Функция сканирует буфер, ища переход из 0 в 1 (ниже порога → выше порога).
После перехода измеряется расстояние до следующего перехода (0→1 или 1→0).
Вычисляется дробная часть distance / markKoef. Если она попадает в интервал [0.25, 0.75] – это считается серединой маркера.

Возвращается расстояние между маркерами (количество битов от начала одного маркера до начала следующего).

5.3 Быстрый поиск (QfindFraseMark)
Проверяет, что начиная с текущей позиции есть widthPartOfMF единиц, а затем widthPartOfMF нулей (или наоборот). Используется для быстрой верификации.

5.4 Сбор слов (FillMasGroup)
Параметры:

countPointToPrevM – расстояние от предыдущего маркера до текущего.

currentMarkFrBeg – позиция текущего маркера в буфере.

orbInf – строка типа "M08", "M04" и т.д.

Алгоритм:

numWordInByte = StrToInt(orbInf[3]) * 2 – количество слов между маркерами (например, для M08 = 16 слов).

stepToTransOrbOne = countPointToPrevM / numWordInByte / 12 – шаг в битах на один бит слова.

Для каждого слова:

Собираются 12 бит, начиная с середины маркера (сдвиг на 4 точки вперёд).

Слово записывается в masGroup (11 младших бит) и masGroupAll (12 бит).

Увеличивается счётчик слов во фразе.

Чётные фразы (2,4,6,…,14) анализируются для поиска маркеров группы/цикла (аналогично M16, но код извлекается из старшего бита слова).

При заполнении группы вызывается OutDate.

6. Адресная система (извлечение параметров)
Адрес имеет формат, например:
M16P1A70B12C10D10T01
M08A1B1T05P05
M16P2A1B1C1T22

6.1 Парсинг
M16, M08 … – информативность.

P1 или P2 – начальное смещение в группе (1 или 2). Для M16 P1 = 1, P2 = 2; для остальных всегда P1.

Далее идут множители A, B, C, D, E, X с коэффициентами K0, K1, K2:

K0 → множитель 8

K1 → множитель 4

K2 → множитель 2
Например, A70 означает A с номером 7 и коэффициентом 0 (т.е. 7*8).

Заканчивается Tnn – тип канала:

T01 – аналоговый (10 бит, смещение 0)

T01-01 – аналоговый (9 бит)

T05 – контактный (битовый), после T05 идёт Pxx – номер бита (1..10)

T21, T22, T24 – быстрые параметры (разной разрядности)

T25 – БУС (CAN‑подобный)

T11 – температурный

6.2 Вычисление позиции в группе
Для M16:

text
numOutElemG = pBeginOffset + 2 * offset
Для остальных:

text
numOutElemG = pBeginOffset + offset
где offset вычисляется как взвешенная сумма множителей с учётом коэффициентов.

Шаг до следующей точки того же адреса:

text
stepOutG = произведение множителей (Fa * Fb * Fc * …)
Для M16 шаг умножается на 2.

6.3 Извлечение значения по адресу
Аналоговый T01 (10 бит): value = masGroup[numOutElemG] >> 1

Аналоговый T01-01 (9 бит): value = masGroup[numOutElemG] >> 2

Контактный T05: из слова выбирается нужный бит (1..10).

Быстрый T21 (8 бит): value = masGroup[numOutElemG] >> 3

Быстрый T22 (6 бит): собирается из двух слов (первое и второе слово по адресу) по специальной таблице.

Быстрый T24 (6 бит): аналогично T22.

БУС T25: из двух последовательных слов (младшее и старшее) формируется 16‑битное значение:
bus = ((highWord & 0x7F8) >> 3) << 8 | ((lowWord & 0x7F8) >> 3).
Маркер БУС – три последовательных значения 65535.

Температурный T11: value = masGroup[numOutElemG] >> 1

7. Формат файла .tlm (телеметрия)
7.1 Заголовок
Текстовый, размером 256 байт, содержит поля вида KEY=VALUE.:

text
Complex=MERATMS-M.SYSTEM=ORBITAIV.
DATA=yyyy:mm:dd.
TIME=hh:mm:sszzz.
OBJ=TMS-M.
SEANS=.
RC=LOC.
MODE=WORK.
T_INP=INT.
INF=16.
INP=RADIO.
FREQ=2299400.
PPU=AUTO.
REF=.
TO=0.0S
Неиспользованные байты заполняются нулями.

7.2 Структура блока данных (один цикл)
Поле	Размер (байт)	Описание
Номер блока	4	порядковый номер цикла (1..4)
Количество слов	4	65536 для M16, 32768 для M08 и т.д.
Время блока (мс)	4	от начала записи
Резерв	4	0
Резерв	4	0
Признак КПСЕВ	4	всегда 0xFFFFFFFF (для M16)
Часы	1	текущий час (0..23)
Минуты	1	
Секунды	1	
Миллисекунды	2	
Резерв (1)	2	0
Признак SEV	1	0
Ошибка	1	0
Резерв (2)	4	0
Данные цикла	переменная	слова по 2 байта (младший байт первым)
Данные цикла – это последовательность 16‑битных слов (12 значащих бит, остальные – служебные).
