# Протокол телеметрии Орбита-IV — разбор для разработчика

## Путь сигнала от датчика до графика

```
Датчик (−2..+8В)
    ↓ напряжение
ЯЛК-96 АЦП (12-бит физически, но в потоке 10-бит для T01)
    ↓ биты
Аналоговый поток (канал E20-10)
    ↓ 12-бит отсчёты АЦП (0..4095)
BitstreamRecoverer: пороговая обработка → биты 0/1
    ↓ кольцевой буфер fifo (2,5 млн бит)
FrameDecoderM16:
    → поиск маркеров фраз → группы → циклы
    → WordProcessing: биты → 12-бит слова → masGroup[]
    ↓
Адресная система: masGroup[i] >> 1 для T01 → code (0..1023)
    ↓
DataPool → getSnapshot() → ChannelValue.value
    ↓
UI: value → вольты (наш слой), отображение, допуски
```

## Структура кадра M16

```
Кадр = 4 цикла
  Цикл = 32 группы
    Группа = 128 фраз = 2048 слов
      Фраза = 16 слов × 12 бит = 192 бита + маркер
        Слово = 12 бит (11 значащих + 1 служебный)
```

Период обновления параметров = длительность одной группы.  
При 10 МГц: ~31 мс (в зависимости от режима).

## Адресная система (для разработчика)

Адрес выглядит так: `M16П1A70B12C10D10T01`

Разбор:
| Часть | Значение |
|---|---|
| `M16` | Информативность: 16 слов во фразе |
| `П1` | Начальное смещение в группе (П1=1, П2=2) |
| `A70` | Множитель A: 7×8=56 (коэф 0→×8) |
| `B12` | Множитель B: 1×4=4 (коэф 2→×2) |
| `C10` | Множитель C: 1×8=8 |
| `D10` | Множитель D: 1×8=8 |
| `T01` | Тип: аналоговый 10-бит |

Позиция в группе:
```
numOutElemG = П + 2 × (A_val×8 + B_val×4 + C_val×8 + D_val×8 + ...)
```
(для M16 умножается на 2)

## Типы каналов и что они значат для нас

| Тип | Бит | Формула | Диапазон | Где применяется |
|---|---|---|---|---|
| `T01` | 10 | `masGroup[i] >> 1` | 0..1023 | Медленные аналоговые (ЯЛК-96, БСИ) |
| `T05` | 1 | `(word >> bitN) & 1` | 0/1 | Контактные (дискретные сигналы) |
| `T21` | 8 | `masGroup[i] >> 3` | 0..255 | Быстрые (ЯТКБСУ, гироскопы) |
| `T22` | 6 | из двух слов | 0..63 | Быстрые двухсловные |
| `T11` | 8 | `masGroup[i] >> 1` | 0..255 | Температурные |
| `T25` | 16 | из двух слов | 0..65535 | BUS (CAN-подобный) |

## Формула код → вольты для T01

Это наш слой (UI), ядро отдаёт только raw code:

```cpp
// Базовая (без калибровки)
double codeToVolts(double code) {
    return code * (10.0 / 1023.0) - 2.0;
}
// code=0    → -2.000 В
// code=512  →  3.004 В  
// code=839  →  6.203 В  (≈ 6.2В калибровочный)
// code=1023 →  8.000 В
```

## Файл .tlm

Заголовок 256 байт текста:
```
Complex=MERATMS-M.SYSTEM=ORBITAIV.
DATA=2026:06:18.
TIME=14:23:05000.
INF=16.
...
```

Затем блоки по одному циклу (65536 слов × 2 байт = 131072 байт):
```
[4 байт] номер блока
[4 байт] кол-во слов (65536)
[4 байт] время (мс)
[4 байт] резерв
[4 байт] резерв
[4 байт] КПСЕВ (0xFFFFFFFF)
[1 байт] часы
[1 байт] минуты
[1 байт] секунды
[2 байт] миллисекунды
[2 байт] резерв
[1 байт] SEV
[1 байт] ошибка
[4 байт] резерв
[65536×2 байт] данные
```

Слова хранятся little-endian, 12 значащих бит из 16.

## Важные нюансы

### Кириллица в адресах
В исходных файлах конфигов используется **кириллица** (М, П — русские буквы). В коде после нормализации (`encoding::normalizeAddress`) они заменяются на латиницу. В нашем коде работаем только с нормализованными адресами.

### "Быстрые" и "медленные" параметры
- **Быстрые (T21)** — обновляются каждую группу (31 мс). Применяются для гироскопов, ЯТКБСУ.
- **Медленные (T01)** — аналоговые параметры ЯЛК-96, БСИ. Те же 31 мс, но важна точность напряжения.

### Два потока (П1, П2)
M16 поддерживает П1 (первый поток) и П2 (второй поток). Разные ячейки занимают разные области пространства адресов.

### Дублирование адресов (_зу)
В Table parameters есть столбцы `*_зу` — это **резервные адреса** (ЗУ = запасное устройство). Для тестирования ЯЛК-96 используем основные адреса.
