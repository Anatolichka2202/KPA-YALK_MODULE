## АРХИТЕКТУРА ТЕСТОВЫХ МОДУЛЕЙ

### Принцип: один API, много драйверов

```
orbita::Orbita API  (заморожен, не трогать)
         │
         │  getSnapshot()
         │  getValueByAddress(addr)
         ▼
  ┌─────────────────────────────┐
  │      TestSession            │  ← общий оркестратор
  │  (берёт из Orbita данные,   │
  │   управляет железом через   │
  │   ITestDriver*)             │
  └──────────┬──────────────────┘
             │
     ┌───────┼───────┐
     ▼       ▼       ▼
 YalkDriver  YavpDriver  YatpDriver
 (IsdClient  (другой     (другой
  + VISA)     драйвер)    драйвер)
```

**Каждый тестовый модуль добавляет свой `ITestDriver`** и интегрируется через общий `TestSession`. Существующий код не меняется — только добавляется.

### T2 — IsdClient (HTTP клиент)

**Суть:** Qt класс для отправки POST запросов к ИСД.

**Интерфейс (минимальный):**
```cpp
class IsdClient : public QObject {
    Q_OBJECT
public:
    explicit IsdClient(const QString& ip = "192.168.0.102", QObject* parent = nullptr);

    // Задать напряжение V Вольт на аналоговый канал ch (1..96)
    // work=true → включить выход
    void setVoltage(int ch, double volts, bool work = true);

    // Установить цифровой канал ch (1..64) в значение 0 или 1
    void setDigital(int ch, bool value);

    // Сброс всех каналов в 0
    void resetAll();

signals:
    void requestFinished(bool ok, const QString& error);
};
```

**Протокол:**
- POST `http://{ip}/`
- Тело: `type=5&num={ch}&val={volts:.6f}&work=1` (тип 5 = напряжение)
- Используй `QNetworkAccessManager` (уже в Qt6, не нужен новый dep)
- Ответ: текстовый "OK" или описание ошибки

**IP — из настроек (QSettings), не хардкодить!**

**Зависимости:** только Qt (QNetworkAccessManager)

**Что проверить вручную перед реализацией:** ping 192.168.0.102, убедиться что ИСД онлайн.

### T3 — VoltmeterClient (VISA клиент)

**Суть:** Qt класс для чтения показаний вольтметра АКИП по NI-VISA.

**Интерфейс:**
```cpp
class VoltmeterClient : public QObject {
    Q_OBJECT
public:
    explicit VoltmeterClient(const QString& visaResource, QObject* parent = nullptr);

    // Подключиться / отключиться
    bool open();
    void close();

    // Измерить DC напряжение (среднее из avgCount измерений)
    // Возвращает INVALID_VOLTAGE (-99.0) при ошибке
    double measureDC(int avgCount = 5);

    // Идентификация прибора (*IDN?)
    QString identify();

signals:
    void error(const QString& msg);
};
```

**Протокол VISA:**
- `*IDN?` → идентификация
- `MEASURE:VOLTAGE:DC?` → текстовый ответ типа "+3.14159265E+00"
- Подключение: через VISA resource string из настроек (напр. `"TCPIP0::192.168.0.XXX::INSTR"` или `"USB0::0x...::INSTR"`)

**Зависимости:** `visa32.dll` / NI-VISA **должна быть установлена на тестовом стенде**
- Линковать через `#pragma comment(lib, "visa32.lib")` или `QLibrary`
- Проверить наличие NI-VISA перед запуском (если нет — graceful disable)

**Альтернатива (если NI-VISA нет):** subprocess к Python + pyvisa — обёртка через `QProcess`, но это план Б.

### T5 — TestSession (оркестратор)

**Суть:** Общий менеджер тестовой сессии. Не зависит от типа ячейки.

**Логика одного шага:**
```
1. driver->applyStimulus(point)    ← задать стимул (ИСД, или другое)
2. sleep(settle_ms)                ← ждать стабилизации (настраиваемо, default 500ms)
3. voltmeter->measureDC(avg)       ← измерить эталон (если voltmeter != nullptr)
4. orbita.waitForData(0)           ← дождаться свежего снимка (уже реализовано!)
5. value = orbita.getValueByAddress(addr) ← прочитать сырой код
6. v_tlm = code * 10.0/1023.0 - 2.0       ← конвертировать (T01 specific)
7. error = |v_tlm - v_ref| / 10.0 * 100   ← погрешность % от шкалы
8. passed = error < tolerance_percent
9. emit pointDone(result)
```

**Данные TestPoint:**
```cpp
struct TestPoint {
    int         isdChannel;      // 1..96 (кто управляет ИСД)
    QString     address;         // телеметрический адрес (нормализованный)
    QString     paramName;       // "Параметр А1"
    double      setVoltage;      // задаём на ИСД (В)
    // --- заполняется во время теста ---
    double      refVoltage;      // вольтметр (эталон)
    double      tlmVoltage;      // телеметрия → вольты
    double      errorPercent;    // погрешность
    bool        passed;
};
```

**Сигналы TestSession:**
```cpp
signals:
    void pointDone(int index, const TestPoint& result);
    void sessionFinished(bool allPassed, int totalPoints, int failedPoints);
    void error(const QString& msg);
```

**После сессии:**
- Вызвать `toleranceResolver.setOverride(addr, tol)` для каждого канала
- Экспортировать `.tol` файл с результатами (Override → .tol)
- Экспортировать отчёт (CSV)

**Зависимости:** `orbita::Orbita`, `ITestDriver`, `VoltmeterClient`, `ToleranceResolver`

### T6 — YalkDriver (драйвер ЯЛК-96)

**Суть:** Реализует `ITestDriver` для ячейки ЯЛК-96. Использует `IsdClient` + `VoltmeterClient`.

**Логика:**
- `setup()` → ping ИСД, идентифицировать вольтметр
- `applyStimulus(point)` → `isdClient.setVoltage(point.isdChannel, point.setVoltage)`
- `teardown()` → `isdClient.resetAll()`

**Тестовые точки (минимальный план):**
```
Для КАЖДОГО из 62 аналоговых каналов:
  - setVoltage = -1.5В
  - setVoltage = 0.0В
  - setVoltage = 2.5В
  - setVoltage = 5.0В
  - setVoltage = 7.5В
```

_Итого: 62 каналов × 5 точек = 310 измерений. При settle_ms=500 ≈ ~3 мин._

**Конвертация кода → вольты (T01):**
```cpp
double codeToVolts(double code) {
    return code * (10.0 / 1023.0) - 2.0;
}
// Проверка: code=839 → ≈ 6.20В (калибровочный канал)
```

### T7 — TestReport (экспорт результатов)

**Суть:** Сохранение результатов в `.tol` файл и CSV отчёт.

**Формат .tol (уже определён в tolerances.md):**
```
# Тест ЯЛК-96, 2026-06-18 14:23:05
# Оператор: ...
# Допуск: 0.5% от шкалы
# addr                    lo       hi       nominal
M16P1A70B12C20D10T01      816.1    824.9    820.5
M16P1A70B12C30D10T01      817.3    826.1    821.7
...
```

**Расчёт lo/hi:**
```cpp
double nominalCode = (refVolts + 2.0) / 10.0 * 1023.0;
double marginCode  = (tolerancePct / 100.0) * 1023.0;  // % от полной шкалы
lo = nominalCode - marginCode;
hi = nominalCode + marginCode;
```

**Формат CSV отчёта:**
```csv
channel,address,param_name,isd_ch,set_v,ref_v,tlm_v,error_pct,passed
1,M16P1A70B12C20D10T01,Параметр А1,1,2.500,2.498,2.503,0.05%,PASS
```

## ГОТОВЫЕ ЗНАНИЯ — формулы и константы

```cpp
// T01 (ЯЛК-96, медленные аналоговые)
double t01_to_volts(int code) {
    return code * (10.0 / 1023.0) - 2.0;
}
// Диапазон: code=0→-2В, code=511→+2.99В, code=839→≈+6.2В, code=1023→+8В

// Погрешность (% от полной шкалы 10В)
double error_pct(double v_tlm, double v_ref) {
    return std::abs(v_tlm - v_ref) / 10.0 * 100.0;
}

// Расчёт допусков для .tol файла (в кодах)
void calc_tolerance(double ref_volts, double tol_pct,
                    double& lo, double& nominal, double& hi) {
    nominal = (ref_volts + 2.0) / 10.0 * 1023.0;
    double margin = tol_pct / 100.0 * 1023.0;
    lo = nominal - margin;
    hi = nominal + margin;
}

// Калибровочный канал (проверка системы)
// addr = "M16P1A70B12C80D80T01"
// При V_ref ≈ 6.2В ожидаем код ≈ 839
```
