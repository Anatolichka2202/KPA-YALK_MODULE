# ИСД — Имитатор Датчиков: архитектура и протокол

## Что такое ИСД

ИСД — **Имитатор Сигналов Датчиков**. Аппаратный стенд на базе микроконтроллера LPC1788 (ARM Cortex-M3), который **создаёт тестовые сигналы** для ячейки БСКУ/ЯЛК вместо реальных датчиков.

```
[ПК — OrbitaDesktop]
        │ POST HTTP
        ▼
  [ИСД: LPC1788]       ← мы управляем по сети
        │ UART 8-байт пакеты
        ▼
  [DM1/DM2/DM3]        ← аппаратные ЦАП-модули (аналог)
  [KM4/KM5]            ← коммутаторы (цифровые каналы)
        │ напряжение -2..+8V / контакты 0/1
        ▼
  [ЯЛК-96]             ← тестируемая ячейка
        │ телеметрия → АЦП → цифровой поток
        ▼
  [E20-10 АЦП]
        │
  [OrbitaDesktop]      ← принимаем обратно и сравниваем
```

## Параметры устройства

| Параметр | Значение |
|---|---|
| Микроконтроллер | NXP LPC1788, ARM Cortex-M3 |
| IP адрес | **192.168.0.102** (из tcpip.h) |
| Порт | **80 (HTTP)** |
| Аналоговых каналов | **96** (–2V до +8V) |
| Контактных каналов | **64** (0 или 1) |
| Связь с модулями | UART, 8-байт пакеты с CRC16 |

## HTTP API ИСД

Все команды — POST-запрос на `http://192.168.0.102/` с телом `application/x-www-form-urlencoded`.

### Тип 5 — установить напряжение (основной для нас)

```http
POST http://192.168.0.102/
Content-Type: application/x-www-form-urlencoded

type=5&num=1&val=3.14159&work=1
```

| Параметр | Описание |
|---|---|
| `type=5` | Задать аналоговый канал в **вольтах** |
| `num=N` | Номер канала ИСД: 1..96 |
| `val=V.VV` | Напряжение в вольтах: **–2.0 до 8.0** |
| `work=1` | Включить канал (work=0 — выключить) |
| `bus=0` | Не подключать к общей шине |

Ответ ИСД: `OK` или текст ошибки.

### Тип 1 — установить код АЦП (сырой)

```
type=1&num=N&val=CODE&work=1
```
`val` = код 0..4095 (12-бит ЦАП).

### Тип 4 — сброс

```
type=4&num=1     // сброс всех каналов
type=4&num=2     // только аналоговые
type=4&num=3     // только контактные
```

### Тип 6 — двухточечная калибровка канала

```
type=6&num=N&p1code=C1&p2code=C2&p1val=V1&p2val=V2
```
Задаёт линейную калибровку для канала N: две точки (код, вольты).

## Формула ЦАП в прошивке ИСД

```c
// type=5 (по напряжению) — из easyweb.c:
val_code = (val_volt + 2.0) * 400;
```

Это значит:
- −2.0 В → код 0
- 0.0 В → код 800
- +6.2 В → код 3280
- +8.0 В → код 4000
- Полная шкала: **10 В** → **4000 единиц** (из 4095 максимум)

**Обратная формула** (код → вольты в ИСД):
```
V = code / 400.0 - 2.0
```

> Примечание: калибровка (type=6) в текущей прошивке **закомментирована**. Используется только упрощённая формула × 400. Это означает, что реальное напряжение на выходе может отличаться от заданного — именно поэтому нужен вольтметр как **эталон**.

## Структура маппинга каналов ИСД

Каналы ИСД нумеруются 1..96, но физически они разбиты по трём аппаратным модулям:

| Модуль | Каналы DM | Каналов | Подключение |
|---|---|---|---|
| DM1 | каналы 1..32 | 32 | ИСД каналы 1,3,5,7,9,12,13,15,16,19,22,24,26,28,30,32,51,53,55,57,59,62,63,65,69,72,74,76,78,80,82 |
| DM2 | каналы 1..32 | 32 | остальные аналоговые |
| DM3 | каналы 1..32 | 32 | остальные аналоговые |

Таблица `DMTable[96][2]` в прошивке задаёт точное соответствие: номер канала ИСД → (адрес DM, номер канала DM).

## Соответствие каналов ИСД и адресов телеметрии

Это **ключевой вопрос**. ИСД канал N физически подключён к датчику ЯЛК-96 → телеметрический адрес M16П1A70B12C...T01.

Соответствие нужно взять из **Приложения 1 (аналог).docx** и **протокол Орбита.pdf** — там дана таблица: номер выхода ИСД → телеметрический параметр → адрес.

Предварительная гипотеза по Table parameters:
- "Параметр А1" = `M16П1A70B12C10D10T01` = ИСД канал 1 (первый аналоговый)
- "Параметр А2" = `M16П1A70B12C20D10T01` = ИСД канал 2
- ...
- "Эталонная калибровка бси 6,2В" = `M16П1A70B12C80D80T01` = ИСД канал 63 (калибровочный, всегда 6.2В)
