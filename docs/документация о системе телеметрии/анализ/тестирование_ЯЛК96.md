# Стратегия тестирования ЯЛК-96

## Контекст: почему раньше было "не совсем верно"

Прежнее понимание предполагало, что допуски (lo/nominal/hi) **заданы заранее** в файле или БД.

**Реальность**: при тестировании ЯЛК-96 нет заранее известного "правильного" значения для каждого канала. Эталон появляется **только в момент измерения** — это показание вольтметра АКИП. Именно это значение и становится "номиналом".

## Полный цикл одного тестового измерения

```
1. ИСД: POST type=5&num=N&val=V&work=1
            ↓ устанавливаем напряжение V на выходе N
            
2. Вольтметр АКИП (VISA):
   MEASURE:VOLTAGE:DC?
            ↓ получаем V_ref = 3.141592 В (эталон, 6+ знаков)
            
3. ЯЛК-96 читает физическое напряжение → АЦП → телеметрия
            ↓
4. E20-10 АЦП → OrbitaDesktop декодирует → raw code (0..1023 для T01)
            ↓
5. Перевод в вольты: V_tlm = code * 10.0 / 1023.0 - 2.0
            ↓
6. Погрешность: error = |V_tlm - V_ref| / 10.0 * 100%
   (от полной шкалы 10В, или от V_ref — настраиваемо)
            ↓
7. Проверка: error < 0.5% → НОРМА / > 0.5% → НАРУШЕНИЕ
```

## Формула перевода кода телеметрии в вольты

### Базовая (без калибровки)

Для канала типа **T01 (10-бит аналог)**:
```
code_tlm  ← raw value из getSnapshot() (0..1023)
V_tlm = code_tlm * 10.0 / 1023.0 - 2.0
```
- code=0 → −2.000000 В
- code=819 → +6.000014 В  
- code=1023 → +8.000000 В

### Откуда −2.0 и ×10?

- Полная шкала ЦАП ИСД: от −2В до +8В = **10В**
- ЯЛК-96 АЦП измеряет этот диапазон: 0..1023 = −2В..+8В
- Линейное отображение: V = code/1023 * 10 − 2

### Калибровочный канал ("Эталонная калибровка бси 6,2В")

В Table parameters есть параметр: `"Эталонная калибровка бси 6,2В"` (адрес `M16П1A70B12C80D80T01`). Этот канал всегда подаёт 6.2В. При тестировании можно использовать его для проверки формулы перевода:
```
expected code для 6.2В = (6.2 + 2.0) / 10.0 * 1023 = 838.86 ≈ 839
```
Если декодированный код для этого канала ≈ 839 — формула верна.

## Архитектура модуля тестирования в OrbitaDesktop

### Новые компоненты

```
desktop_orbita/
├── test_session/
│   ├── test_session.h/.cpp      ← менеджер тестовой сессии
│   ├── isd_client.h/.cpp        ← HTTP клиент для ИСД
│   ├── voltmeter_client.h/.cpp  ← VISA клиент для АКИП
│   ├── test_plan.h/.cpp         ← план теста (список точек)
│   └── test_result.h/.cpp       ← результат с протоколом
```

### IsdClient — минимальный API

```cpp
class IsdClient : public QObject {
    Q_OBJECT
public:
    IsdClient(const QString& ip = "192.168.0.102", QObject* parent = nullptr);

    // Основная операция — задать напряжение
    QFuture<bool> setVoltage(int channel, double volts, bool enable = true);
    
    // Сброс всех каналов
    QFuture<bool> resetAll();
    
    // Контактный канал
    QFuture<bool> setDigital(int channel, bool value);

signals:
    void error(const QString& msg);
};
```

Запрос: `POST http://192.168.0.102/` body: `type=5&num={ch}&val={v:.6f}&work=1`

### VoltmeterClient — чтение АКИП по VISA

```cpp
class VoltmeterClient : public QObject {
    Q_OBJECT
public:
    // VISA resource string, например "TCPIP::192.168.0.XXX::INSTR"
    VoltmeterClient(const QString& visaResource, QObject* parent = nullptr);
    
    QFuture<double> measureDC();    // MEASURE:VOLTAGE:DC?
    QFuture<QString> identify();    // *IDN?
};
```

### TestSession — оркестратор

```cpp
struct TestPoint {
    int         isdChannel;    // 1..96
    QString     tlmAddress;    // M16П1A70B12...T01
    double      setVoltage;    // что задаём на ИСД (В)
    double      refVoltage;    // что измерил вольтметр (заполняется)
    double      tlmVoltage;    // что пришло из телеметрии (заполняется)
    double      errorPercent;  // |tlm - ref| / 10.0 * 100
    bool        passed;        // < tolerance
};

class TestSession : public QObject {
public:
    void setTolerance(double percent);  // default 0.5
    void setSettleTimeMs(int ms);       // пауза после установки (default 500)
    
    void run(const QList<TestPoint>& plan);
    
    // Экспорт результатов в .tol файл (Override layer)
    void exportToTolFile(const QString& path);
    
    // Экспорт протокола
    void exportReport(const QString& path);

signals:
    void pointDone(int index, const TestPoint& result);
    void finished(bool allPassed);
};
```

## Связь с системой допусков

После тестовой сессии результаты **записываются в Override слой** `ToleranceResolver`:

```cpp
// После получения V_ref от вольтметра:
double nominalCode = (ref_volts + 2.0) / 10.0 * 1023.0;
double margin      = 0.5 / 100.0 * 10.0 / 10.0 * 1023.0;  // 0.5% от шкалы в кодах

chstatus::Tolerance tol;
tol.set     = true;
tol.nominal = nominalCode;
tol.lo      = nominalCode - margin;
tol.hi      = nominalCode + margin;
toleranceResolver.setOverride(tlmAddress, tol);
```

И сохраняются в `.tol` файл для следующих сессий:
```
# Сессия тестирования 2026-06-18 14:23
# Канал ИСД 1 → M16П1A70B12C10D10T01
# V_ref=3.141592, V_tlm=3.139, error=0.08%
M16П1A70B12C10D10T01 816.1 824.9 820.5
```

## Параметры погрешности

| Параметр | Значение | Описание |
|---|---|---|
| `tolerance_percent` | 0.5 | % от полной шкалы (настраиваемо) |
| `settle_time_ms` | 500 | Пауза после установки ИСД перед измерением |
| `avg_count` | 3..10 | Усреднение показаний вольтметра |
| `avg_count_tlm` | 3..5 | Усреднение кодов телеметрии |

## Тестовый план (минимальный для ЯЛК-96)

| # | Напряжение ИСД | Назначение |
|---|---|---|
| 1 | −1.5 В | нижняя зона шкалы |
| 2 | 0.0 В | ноль |
| 3 | 1.0 В | |
| 4 | 2.5 В | середина |
| 5 | 4.0 В | |
| 6 | 5.0 В | |
| 7 | 6.2 В | совпадает с калибровочным каналом |
| 8 | 7.5 В | верхняя зона |

Для каждого напряжения тестируются **все 62 аналоговых канала** ЯЛК-96.

## Что НЕ входит в текущую задачу

- Другие ячейки (ЯФК, БСУФТК) — отдельные задачи, протокол связи одинаковый
- Контактные параметры (T05) — отдельный модуль тестирования
- Быстрые параметры (T21, ЯТКБСУ) — другой диапазон и другой ИСД канал

## Зависимости в коде

- `IsdClient` ← `QNetworkAccessManager` (уже в Qt)
- `VoltmeterClient` ← `visa32.dll` / `NI-VISA` — нужно наличие NI-VISA на стенде
- `TestSession` ← `ToleranceResolver`, `orbita::Orbita::getValueByAddress`
