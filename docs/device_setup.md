# 🖥️ Режимы работы
- Headless (встраиваемый) – библиотека не создаёт своих потоков (кроме потока захвата, но он управляется через start/stop). Внешняя система вызывает orbita_wait_for_data в своём цикле.

- Серверный – внутри библиотеки запускается std::thread с HTTP-сервером (на cpp-httplib). Пользователь может отправлять команды через REST. Этот режим включается отдельной функцией orbita_start_server(port).

- Desktop – отдельное Qt-приложение, которое подключается к библиотеке статически или динамически, использует её API и отображает графики.

# 🐍 Python для анализа больших данных
#### Библиотека предоставляет экспорт данных в CSV и бинарный формат TLM. 
### Для анализа больших объёмов можно использовать Python (pandas, numpy) – это будет отдельный инструмент, не входящий в библиотеку. Но сам модуль остаётся на C++ для производительности и интеграции.
