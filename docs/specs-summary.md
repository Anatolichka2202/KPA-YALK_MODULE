## Ключевые открытия (то, что изменило архитектуру)

### 1. Допуски появляются только при измерении

Прежнее понимание: lo/nominal/hi задаются заранее в БД или .tol файле.

**Реально**: эталонные значения — это показания вольтметра АКИП в момент тестирования. `.tol` файл — **выход** тестовой сессии, не её входные данные.

Это меняет ToleranceResolver: Override-слой заполняется **во время теста** → затем сохраняется в `.tol`.

### 2. ИСД управляется по HTTP

ИСД (Имитатор Датчиков) — LPC1788, IP 192.168.0.102.
Установка напряжения: `POST /` body `type=5&num=N&val=V.VV&work=1`.
Нет REST JSON — простой URL-encoded POST.

### 3. Цепочка измерения для ЯЛК-96

```
ИСД (POST -2..+8В) → физический выход → ЯЛК-96 АЦП → телеметрия
                   → Вольтметр АКИП (VISA) → V_ref (эталон)
Сравнение: V_tlm vs V_ref, погрешность < 0.5% (настраиваемо)
```

## ИСД — Имитатор Датчиков

- **Железо**: NXP LPC1788, ARM Cortex-M3
- **IP**: `192.168.0.102:80`
- **96 аналоговых каналов**: –2В до +8В (полная шкала 10В)
- **64 контактных канала**: 0 или 1
- **Формула ЦАП**: `code = (volts + 2.0) × 400` (12-бит, 0..4095)
- **Маппинг на оборудование**: UART 8-байт пакеты с CRC16 → DM1/DM2/DM3 (аналог), KM4/KM5 (цифра)

Подробнее: [`анализ/ИСД_архитектура.md`](документация%20о%20системе%20телеметрии/анализ/ИСД_архитектура.md)

## Протокол телеметрии

- **Режим M16**: 16 слов/фраза, 128 фраз/группа, 32 группы/цикл, 4 цикла/кадр
- **Период обновления**: ~31 мс (при 10 МГц E20-10)
- **Тип T01** (аналоговый 10-бит): `value = masGroup[i] >> 1` → 0..1023
- **Перевод в вольты**: `V = code × 10/1023 − 2.0`
- **Адреса**: `M16П1A70B12C10D10T01` — кириллица нормализуется перед передачей в ядро

Подробнее: [`анализ/протокол_телеметрии.md`](документация%20о%20системе%20телеметрии/анализ/протокол_телеметрии.md)

## База данных параметров

- **Table parameters.md** (146KB): ~1000+ параметров с адресами и категориями
- Категории: "БСИ МКА-1 аналоговые" (A1..A62), "Высокочастотные и быстроменяющиеся" (B1..B16), "Служебные", "БСИ МКС-1 контактные" (K1..K64)
- **Калибровочный канал**: "Эталонная калибровка бси 6,2В" = `M16П1A70B12C80D80T01` → код ≈ 839
- **Резервные адреса** (_зу): каждый параметр имеет дублёр — для тестирования используем основные

## Рекомендации по API (ux_and_api.md)

- `ChannelRegistry` с RCU-паттерном для безопасной замены конфига без остановки
- `getSnapshot()` — атомарный срез всех значений
- `pause()`/`resume()` для заморозки при анализе
- `setDataCallback(cb)` для RuleEngine (тревоги)
- UX: ≤3 клика, статус всегда виден, нет модальных диалогов во время сбора
